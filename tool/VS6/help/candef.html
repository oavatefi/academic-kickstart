CAN configuration

how to configure CAN signales

To configure CAN data edit an *.ini file. As default VS6 uses "candef_default.ini". You can copy and change it.
You can easily take over the configuration directly from a Vector dbc file.
A CAN signal can be configured by following values:

[odom_x_mm]
ID = 0x65D
strbit = 32
length = 16
byteorder = 0
valuetype = 1
factor = 1
offset = 0
muxed = 0
multiplexor = 0

ID: CAN ID (hex); for extended IDs edit out "x" (e.g. "0x1E7CA012" not "0x1E7CA012x")
strbit: Start bit
length: Signal length (bit)
byteorder: 0 = Intel, 1 = Motorola
valuetype: 0 = unsigned, 1 = signed
factor: factor of signal
offset: offset of signal
muxed: 0 = signal is not multiplexed, 1 = signal is multiplexed (multiplexor needed)
multiplexor: value of multiplexor (e.g. 0x1)

how to configure CAN multiplexed signales

With VS6II you can define 4 multiplexor signals for 4 multiplexed messages.
The multiplexor signal can be configured like a normal signal. The difference is to define "isMultiplexor":

ID = 0x68A
strbit = 0
length = 8
byteorder = 0
valuetype = 0
factor = 1
isMultiplexor = 1   <-- Signal is an multiplexor!

The multiplexor signals are:

[multiplexor000]
[multiplexor001]
[multiplexor002]
[multiplexor003]

If a signal is multiplexed it must be defined as multiplexed signal (muxed = 1) and the number of the multiplexor must be set (multiplexor).
(multiplexor is not the number of the multiplexor signal e.g. multiplexor001, it is the value of the multiplexor.)

[sl_mm]
ID = 0x68B
strbit = 40
length = 15
byteorder = 0
valuetype = 0
factor = 1
muxed = 1   <-- Signal is multiplexed
multiplexor = 0x4   <-- multiplexor value is 4
